# Sonartra Six-Report Suite Map

## 1. Suite Purpose

The Sonartra assessment suite is a behavioural operating system for work. It helps a reader understand the patterns that shape how they lead, decide, respond to pressure, communicate, create value, and sustain performance.

The suite should not feel like six versions of the same leadership report. Each assessment must own a different workplace question and produce a premium report that is useful on its own while becoming more valuable when read alongside the others.

The full suite helps the reader see:

- how they create direction and mobilisation through others
- how they reach decisions when trade-offs are real
- how their behaviour changes under pressure
- how they create understanding, influence, and trust
- where they naturally create organisational value
- how they sustain focus, energy, execution, and recovery over time

## 2. Suite Design Principles

Every assessment in the suite must follow the report-first fully authored model:

- one active domain per assessment
- four scored signals
- 24 ranked signal patterns
- 24 fully authored premium reports
- one report per `pattern_key`
- no score-shape report variants
- no modular report-language worksheet assembly
- no runtime AI-generated prose
- deterministic scoring from option weights
- persisted `canonical_result_payload` rendered at runtime

Commercial and quality principles:

- Each assessment must answer a workplace question that is distinct enough to justify its place in the paid suite.
- Each report must make the reader feel recognised without drifting into generic personality-test language.
- Each signal set must be strong enough to support 24 meaningfully different reports.
- Each assessment must reach at least 8.5/10 before monetisation.
- The target suite average should be 9.0/10+ before broader paid traffic.
- Leadership Approach is the benchmark for structure, result persistence, browser readiness, and paid-facing presentation quality.

### Assessment 2 - Decision Style

assessment_title: Decision Style

assessment_key: `decision-style`

domain_key: `decision_style`

target_reader_question: How do I decide when the answer is not obvious?

four_signal_model: Evidence, Instinct, Principle, Pragmatism.

signal_1_label: Evidence

signal_2_key: `instinct`

signal_2_label: Instinct

signal_2_definition: Uses judgement, experience, pattern recognition, and felt sense when the data is incomplete.

signal_3_key: `principle`

signal_3_label: Principle

signal_3_definition: Tests decisions against standards, values, obligations, and what should remain true under pressure.

signal_4_key: `pragmatism`

signal_4_label: Pragmatism

signal_4_definition: Narrows toward what will work, what can be executed, and what moves the situation forward.

distinction_from_leadership_approach: Leadership Approach explains how the reader mobilises others after or around direction. Decision Style explains how the reader chooses between options before action is taken.

distinction_from_other_suite_reports: It is about judgement and trade-offs, not communication, pressure behaviour, value creation, or operating cadence.

commercial_value: Very high. Decision quality is a senior workplace concern and produces practical paid value for leaders, founders, product teams, and operators.

- `evidence_pragmatism_principle_instinct`

- `evidence_principle_instinct_pragmatism`
- `instinct_principle_pragmatism_evidence`

authoring_risks: Evidence and Pragmatism can blur if evidence is written as "what works." Principle can become too moralising if not kept workplace-specific.

quality_bar: Should reach 8.8+ before paid release because it is the strongest candidate for the second commercial report.

## 6. Signal Architecture Review

Leadership Approach: Results, Process, Vision, and People are mutually distinct and already proven through the live report-first model. The main risk is breadth: each signal can touch other suite domains, so Leadership must keep them framed as leadership mobilisation signals.

Decision Style: Evidence, Instinct, Principle, and Pragmatism are commercially strong and capable of producing 24 useful patterns. Evidence and Pragmatism need careful distinction: evidence is about what is known; pragmatism is about what will work. Principle must be written as standards and obligations, not moral superiority.

Pressure Pattern: Control, Adaptation, Drive, and Composure are distinct and workplace-useful. The set can support rich patterns because each signal responds to pressure differently. The risk is therapy-coded language; the report must stay behavioural, not diagnostic.

Communication Style: Clarity, Empathy, Challenge, and Narrative are understandable and strong enough for 24 reports. Challenge is important because it prevents the model from becoming simply "clear and nice." Narrative must not become Leadership's Vision signal; it should mean how ideas are structured and carried through communication.

Value Creation: Growth, Efficiency, Quality, and Relationships are commercially legible. The signal set can sustain 24 patterns, but the authoring must avoid generic business jargon. Relationships must be framed as organisational value, not social preference.

Performance Rhythm: Focus, Momentum, Flexibility, and Renewal are distinct but need the most careful positioning. Renewal could become wellness-coded; it should mean capacity management and recovery in service of sustained work. The set is viable if written as operating cadence.

## 8. Suite-Level Anti-Overlap Rules

- Leadership Approach owns how someone leads through others; Decision Style owns how they choose between options.
- Leadership Approach owns mobilisation and direction; Communication Style owns how meaning and influence are transmitted.
- Pressure Pattern owns behaviour under constraint; Performance Rhythm owns sustainable operating cadence.
- Value Creation owns where someone creates organisational value; Leadership Approach owns how they lead people toward outcomes.
- Decision Style owns judgement before commitment; Value Creation owns the type of contribution created by the work.
- Communication Style owns message, audience, trust, and challenge; Decision Style owns decision inputs and trade-offs.
- Performance Rhythm owns ongoing capacity and cadence; Pressure Pattern owns acute behaviour when stakes or constraints rise.
- Value Creation must not become a career archetype model. It should stay focused on contribution patterns.
- Pressure Pattern must not become a wellbeing report. It should stay focused on observable workplace behaviour under pressure.
- Communication Style must not become presentation skills. It should explain the reader's communication pattern across real workplace moments.

## 9. Recommended Authoring Order

Recommended sequence after Leadership Approach:

1. Decision Style
2. Pressure Pattern
3. Communication Style
4. Value Creation
5. Performance Rhythm

Rationale:

Decision Style should come second because it has the clearest commercial value, the strongest buyer-recognised use case, and the most distinct separation from Leadership Approach. It is also likely to produce strong paid-reader recognition quickly.

Pressure Pattern should come third because it is commercially useful and emotionally resonant, but it needs careful language control to avoid therapy-coded content.

Communication Style should come fourth because it is broadly valuable, but it risks generic workplace-communication language unless the signal model is sharpened through Decision and Pressure learnings.

Value Creation should come fifth because it could become a powerful executive/founder report, but its positioning needs more strategic precision.

Performance Rhythm should come sixth because it is useful but has the highest risk of drifting into wellness or productivity content unless the suite voice is already well established.
