Sonartra MVP Web App — Build Task Roadmap

Purpose:
This document defines the full sequential build plan for the Sonartra MVP web application.
It is the master execution roadmap for all Codex tasks.

The system must be built engine-first, with a single canonical runtime and result contract.

--------------------------------------------------
PHASE 0 — FOUNDATION AND REPO SETUP
--------------------------------------------------

Task 1 — Bootstrap Sonartra MVP Engine App
- Initialize Next.js (App Router) + TypeScript app
- Setup base project structure
- Configure linting, formatting, env handling
- Create engine-first folder structure
- Ensure build passes

Task 2 — Define canonical engine architecture
- Define engine boundaries
- Define canonical result contract
- Define lifecycle states
- Define service/repository structure
- Define admin vs user vs marketing separation

Task 3 — Create initial app routing structure
- Scaffold route groups:
  - public (marketing)
  - user app
  - admin app
  - admin assessments module
- Create placeholder pages

--------------------------------------------------
PHASE 1 — DATABASE AND ENGINE KERNEL
--------------------------------------------------

Task 4 — Design and implement MVP database schema
- assessments
- assessment_versions
- domains
- signals
- questions
- options
- option_signal_weights
- attempts
- responses
- results

Task 5 — Seed flagship assessment (WPLP-80)
- Insert structured seed data into DB
- Maintain deterministic ordering and IDs
- Ensure all relationships resolve

Task 6 — Build engine core types
- assessment definition
- runtime question model
- response set
- score result
- normalized result
- result payload
- readiness status

Task 7 — Build assessment definition repositories
- load published assessment
- load domains, signals, questions, options, weights

Task 8 — Build runtime execution model loader
- assemble fully resolved runtime definition
- ensure no missing references

--------------------------------------------------
PHASE 2 — SCORING AND RESULT GENERATION
--------------------------------------------------

Task 9 — Build raw scoring engine
- aggregate option → signal weights
- deterministic scoring only

Task 10 — Build normalization engine
- convert scores to percentages
- produce rankings
- ensure consistency and stability

Task 11 — Build canonical result payload
- define single result structure
- include all required sections

Task 12 — Build readiness validator
- enforce strict "ready" criteria
- reject incomplete or invalid results

Task 13 — Add engine unit tests
- scoring correctness
- normalization correctness
- payload structure
- readiness validation

--------------------------------------------------
PRIORITY RULE
--------------------------------------------------

Build in this order:

1–18 → core engine
19–24 → user flow
33–36 → stability
25–32 → admin
37+ → marketing

Do NOT build UI or admin features before engine core is proven.
