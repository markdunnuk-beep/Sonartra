# 08 Live Preview QA Prompt

Use this prompt to evaluate a selected result in `/draft-result`.

## Review Surfaces

Check:

- desktop
- mobile
- light mode
- dark mode
- focus mode
- mobile section navigator
- pattern signature
- reading rail
- section rhythm
- content density
- commercial readiness

## Questions

Answer:

- Does the result read clearly without schema knowledge?
- Does the first viewport give useful context?
- Does the pattern feel recognisable?
- Does each section have a distinct job?
- Is the mobile reading experience dense but manageable?
- Is the application guidance practical?
- Does anything feel like filler, hype, or generic coaching?

## Output

Return:

- PASS / NEARLY READY / FAIL
- severity-ranked findings
- exact affected section
- exact weak line where relevant
- recommended targeted fix
- final readiness judgement
